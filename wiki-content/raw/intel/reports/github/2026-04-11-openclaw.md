## 深度分析

### 起源与演变

- **2025年11月**: Peter Steinberger 在马德里开发周末项目 "Clawdbot"，连接 Claude 到 WhatsApp
- **2026年1月**: 因商标纠纷更名为 "Moldbot"，3天后定名 "OpenClaw"
- **2026年1月30日**: 9,000 Stars，发布首日
- **2026年2月15日**: 创建者加入 OpenAI，黄仁勋公开背书
- **2026年3月4日**: 超越 React（250K Stars）
- **2026年3月24日**: 335,000+ Stars

### 架构设计

```
OpenClaw = LLM + 本地操作系统 + 多通信渠道
         = 自动化总线 + 多 Agent 架构 + Markdown 配置
```

**核心能力**:
1. **Skill 系统**: 基于 Markdown 的能力配置，支持多 Agent 协作
2. **记忆机制**: 纯文本日志 + 向量搜索数据库
3. **多模型路由**: 默认 GPT-5.40、Claude Opus 4.60、Gemini 2.5 Flash
4. **48小时超时**: 支持大规模后台任务
5. **可插拔沙箱**: SSH 和 OpenShell 后端

### 安全风险（不可回避）

| 风险类型 | 数据 | 来源 |

| CVE 披露 | 9个 | GrowExx |

**对比自托管 vs 托管平台**:

| 风险领域 | 自托管 | 托管平台 |
|----------|--------|----------|
| 网络隔离 | 手动 VPN + 防火墙 | 私有云，默认 localhost |
| Skill 审计 | DIY（10,700+ skills）|  curated + 多阶段扫描 |
| 提示注入 | 部分识别 | AI 行为过滤 |
| 身份文件保护 | chmod + 监控 | 不可变配置 + 完整性检查 |
| 合规 | 从零构建 | 内置审计日志 |
